The biggest issue isn't the French—it's that the model assumed there is a single translation for **straight**, when English has many meanings.

---

## ❌ Problem 1: It didn't ask "straight in what sense?"

"Straight" can mean:

* a straight line
* go straight ahead
* straight hair
* a straight answer
* heterosexual
* neat / direct
* immediately ("go straight home")
* undiluted alcohol ("straight whiskey")

A good tutor should acknowledge the ambiguity.

For example:

> The English word **straight** has several meanings. Here are the most common translations.

---

## ❌ Problem 2: The definition is circular

It says:

> signifie droit.

That teaches nothing.

Instead:

```markdown
## Meaning

**droit / droite**

Not curved or bent.

Example:

Une ligne droite.
```

---

## ❌ Problem 3: Grammar

It says

> Droit est un adjectif.

That's true.

But it's missing something important.

**Droit** is also a noun.

Examples:

* le droit = law / rights
* avoir le droit de = to be allowed to

Since it's a vocabulary tutor, it should mention that these are different words with different meanings.

---

## ❌ Problem 4: Notes

It says

> utilisé pour les noms masculins...

I'd rather teach agreement.

Example:

```markdown
L'adjectif s'accorde avec le nom.

un chemin droit

une ligne droite
```

Much more useful.

---

## ❌ Problem 5: Synonyms

This is the weakest part.

> Droit peut aussi signifier droit.

That's basically nonsense.

---

## Here's the answer I'd want

For a beginner:

```markdown
# Word

straight

# Most common translations

- droit (masculine)
- droite (feminine)

# Part of speech

Adjective

# Meaning

Describes something that is not curved or bent.

# Agreement

The adjective agrees with the noun.

- un chemin droit
- une ligne droite

# Examples

La route est droite.

Va tout droit.

Trace une ligne droite.

# Other meanings

Depending on the context, "straight" can also mean:

- tout droit (straight ahead)
- hétérosexuel (sexual orientation)
- franc (honest/direct)

The correct translation depends on the sentence.
```

## I think this reveals another improvement to your architecture

I would split `VocabularyPrompt` into **two distinct modes**:

1. **Single word lookup**
2. **Expression lookup**

Those are different tasks.

### Single word

Example:

```
straight
```

or

```
comprendre
```

The prompt can say:

> If the word has multiple common meanings, present them in order of frequency and explain that the correct translation depends on context.

### Expression

Example:

```
ce n'est pas compris
```

Now you explain the phrase as a whole instead of trying to define each word independently.

That distinction is easy to make in Ruby (a single token vs. multiple tokens) and will produce noticeably better vocabulary explanations.
